User Flow (High Level)

Flow A: Plan conflict resolver
1. User sees rule conflict banner on front page.
2. User opens Plan conflicts.
3. System lists conflicts with short explanations.
4. User selects a conflict to view recommended fix.
5. User applies fix and receives confirmation.

Flow B: Registration task hub
1. User opens Registration tasks from front page.
2. System groups tasks by status (Open / Waiting / Confirmed).
3. User selects a task with an open deadline.
4. User registers and sees the task move to Confirmed.

Flow C: Smart course search + plan preview
1. User searches for a course with simplified filters.
2. System shows results and a plan preview panel.
3. User previews schedule impact and sees conflicts.
4. User adds course to plan or goes to conflict resolver.
